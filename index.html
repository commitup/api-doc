    This API serves as the primary gateway to facilitate money transfers through PayPorter platform.

    Transfer

    Transfer is the main resource of the API. It represents a money transfer request. It is created by the sender and then it is processed by the PayPorter platform. Validation is needed to be done before the transfer is created. After validation is done, confirm is need to be called to create the transfer.

    In validations reponse header, you will get and operation-id. This operation-id is needed to be used in confirm request.

    Here are the key points about the API and its transfer types:

    • The API allows for money transfers to be made to other people.
    • There are three types of transfer: by name, bank account, or electronic wallet.
    • The recipient is identified using their name, bank account details, or wallet details, respectively.
    • Before a transfer is created, it must be validated.
    • The confirm method must be called to create the transfer after it has been validated.
    • In the response header of the validate method, you will receive an operation-id which is needed for the confirm request.

    Here are the steps involved in using the money transfer API:

    1. The sender creates a transfer request using the POST method and the appropriate endpoint for the transfer type (e.g. /mt-api/V2/moneysend/to-name/validate for a transfer to name).
    2. The sender includes the necessary parameters in the request body, including the Person objects for the sender and recipient, as well as the transfer amount and other details.
    3. The sender includes the authentication token in the request header (e.g. Authorization: Bearer {auth-token}).
    4. The API processes the request and returns a response indicating whether the request was successful or not.
    5. If the request was successful, the API will return an operation-id in the response header.
    6. The sender uses the operation-id to make a confirm request using the POST method and the appropriate endpoint (e.g. /mt-api/V2/moneysend/confirm).
    7. The sender includes the operation-id in the request header (e.g. operation-id: abc123).
    8. The API processes the confirm request and creates the transfer

    • The API allows for payments to be made from other people.
    • A payment is processed with a reference number from external firm
    • Before a payment is created, it has to be validated.
    • The confirm method has to be called to pay transfer after it has been validated.
    • In the response header of the validate method, you will receive an operation-id which is needed for the confirm request.
